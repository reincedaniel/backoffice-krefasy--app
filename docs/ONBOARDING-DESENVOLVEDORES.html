Krefasy Backoffice — Guia Técnico para Desenvolvedores

Versão: 1.0  |  Data: Julho 2026  |  Público: Novos desenvolvedores

1. Visão geral

O Krefasy Backoffice é uma aplicação web administrativa para gestão de créditos e empréstimos. Permite que administradores gerem clientes, empréstimos, parcelas, configurações de produtos financeiros e atendimento por chat.

4. Arquitetura em camadas

Views (pages) → krefasy-chat.vue, customers/index.vue Components → PageHeader, Modals Stores (Pinia) → krefasy.store.ts Services → messages.service.ts, loans.service.ts API base (Axios) → api.ts (JWT automático) Backend .NET → credit-api.krefasy.com

Fluxo: View → Store/Service → api.ts → Backend → resposta { succeeded, data, message, errors } → Store atualiza refs → View re-renderiza.

6. Autenticação

  • Login: /auth/login → POST /auth/login
  • Sessão: localStorage chave USER_LOGIN (token + user)
  • Rotas protegidas: meta.requiresAuth: true no router
  • Interceptor: adiciona Authorization: Bearer; 401 → redirect login
  • Logout: krefasyStore.logout()

7. Integração API (.NET)

O backend usa PascalCase em query params e multipart. O frontend usa camelCase; os services fazem o mapeamento.

FrontendAPI (.NET)

Envelope de resposta: { "succeeded": true, "data": {...}, "message": "...", "errors": null }

13. Setup local

yarn install
cp env.example .env
yarn dev          # http://localhost:5173
yarn build        # gera dist/
yarn preview      # preview do build

14. Deploy (Docker)

yarn docker:build
yarn docker:push:hub

O Dockerfile serve ficheiros estáticos de dist/ via nginx.
